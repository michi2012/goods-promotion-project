# 데이터 분석 작업 매뉴얼

## 0. 작업 시작 전 필수 확인 사항

### 0-1. 데이터 출처와 신뢰도 확인
- 데이터 출처가 무엇인가?
- 언제 추출된 데이터인가? 최신성이 중요한 분석인가?
- 누가 정제했는가? 이미 가공된 데이터인가, 원본인가?

### 0-2. 데이터 라이센스 / 민감 정보 확인
- 개인정보가 포함되어 있는가?
- 외부 공유 가능한 데이터인가?

### 0-3. 분석 목표 명확화
- 누가 결과를 볼 것인가?

- 정확도가 더 중요한가, 속도가 더 중요한가?

## 1. 데이터 무결성 절대 규칙

### 1-1. 원본 데이터 보호
- **원본 파일을 절대 직접 수정하지 않는다.** 항상 복사본을 만들거나 새 파일에 저장한다.
- 원본 위치, 파일명, 추출 일자를 README 또는 노트북 상단에 기록한다.

### 1-2. 데이터 탐색 우선
어떤 분석/변환을 하기 전 반드시 다음을 확인한다:
```python
df.shape          # 행/열 개수
df.dtypes         # 각 컬럼의 자료형
df.head()         # 첫 몇 줄
df.tail()         # 끝 몇 줄
df.describe()     # 수치형 요약
df.isna().sum()   # 컬럼별 결측치
df.duplicated().sum()  # 중복 행
```

이 결과를 보고하지 않고 분석을 진행하지 마라.

### 1-3. 결측치 처리
- 결측치가 발견되면 사용자에게 보고하고 처리 방침을 묻는다:
    - 행 제거
    - 평균/중앙값/최빈값 대체
    - 보간(interpolation)
    - 별도 카테고리로 처리
- 처리 방침과 그 근거를 코드 주석으로 남긴다.

### 1-4. 이상치 처리
- 이상치 자동 제거 금지. 항상 사용자 확인 후.
- 이상치가 진짜 오류인지 의미 있는 극단값인지 구분.

## 4. 시각화

### 4-1. 차트 필수 요소
모든 차트는 다음을 포함한다:
- 제목 (무엇을 보여주는가)
- X축 레이블 + 단위
- Y축 레이블 + 단위
- 범례 (여러 시리즈일 때)
- 출처/주석 (필요시)

### 4-2. 차트 선택
- 시계열 → 선 그래프
- 카테고리 비교 → 막대 그래프
- 분포 → 히스토그램, 박스플롯, 바이올린 플롯
- 상관관계 → 산점도, 히트맵
- 비율 → 막대 그래프 (파이 차트는 가급적 피하기)

### 4-3. 색상 사용
- 색맹 친화적 팔레트 사용.
- 색상만으로 정보를 전달하지 마라. 모양/패턴/레이블 병행.

### 4-4. 오해를 부르는 시각화 금지
- Y축 0부터 시작 (막대 그래프의 경우)
- 축 자르기를 할 때는 명시
- 3D 차트 가급적 회피

## 5. 분석 결과 검증

### 5-1. 놀라운 숫자 의심
"이상하게 너무 좋다" 또는 "너무 극단적이다" 싶은 결과는 반드시 다른 방법으로 교차 검증.
- 다른 방식으로 같은 숫자 계산
- 샘플 데이터로 수동 검증
- 도메인 지식과 부합하는지 확인

### 5-2. 통계적 검정의 함정
- p-value를 맹신하지 마라. 효과 크기와 신뢰구간도 함께.
- 다중 비교 보정 고려.
- 표본 크기가 충분한가?

### 5-3. 인과 ≠ 상관
상관관계를 인과관계로 보고하지 마라. 명시적으로 구분한다.
